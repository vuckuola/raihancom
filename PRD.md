# PRD + Copy Website — Raihan Com

## 1. Product Overview

### Nama
**Raihan Com**

### Jenis website
Static single-page website untuk GitHub Pages.

### Tujuan utama
Raihan Com butuh website profesional untuk menjelaskan layanan menerima / jual-beli alat bekas kantor, dengan kontak cepat via WhatsApp.

### Layanan inti
Raihan Com menerima alat-alat bekas kantor, terutama:

- Panasonic PABX
- Printer
- CPU / desktop tower
- Proyektor
- HT / radio komunikasi
- Perangkat elektronik kantor lain
- Rak / unit perangkat elektronik kantor bekas

### Kontak utama
- E. J. / PIC disamarkan
- WhatsApp: `[nomor disamarkan]`
- WhatsApp: `[nomor disamarkan]`

### Masalah pengguna
Pemilik kantor, teknisi, admin, atau pengelola aset sering punya alat kantor bekas yang sudah tidak dipakai. Mereka butuh pembeli / penerima yang bisa dihubungi cepat, jelas menerima kategori barang tertentu, dan tampak aman dipercaya.

### Solusi website
Website satu halaman, mobile-first, bersih, elegan, fokus ke:

- Barang apa yang diterima
- Cara menghubungi
- Proses sederhana
- Kepercayaan lewat informasi jelas, bukan klaim kosong
- CTA WhatsApp terlihat sejak awal

---

## 2. Target Users

### 1. Admin kantor
Punya tugas merapikan gudang, mengeluarkan aset lama, atau mencari pihak yang menerima barang bekas kantor.

Kebutuhan:
- Daftar barang diterima jelas
- Kontak cepat
- Bahasa mudah dipahami
- Tidak perlu isi form panjang

### 2. Pemilik usaha / UMKM
Punya printer, CPU, PABX, proyektor, atau perangkat kantor lama.

Kebutuhan:
- Bisa tanya estimasi
- Bisa kirim foto barang via WhatsApp
- Proses tidak rumit

### 3. Teknisi kantor / IT support
Menangani perangkat bekas dari kantor klien.

Kebutuhan:
- Kategori teknis jelas
- Bisa komunikasi langsung
- Nomor WhatsApp aktif mudah ditemukan

### 4. Pengelola aset / gudang
Menangani banyak unit perangkat bekas.

Kebutuhan:
- Bisa bahas jumlah banyak
- Bisa kirim daftar barang
- Website tampak resmi dan rapi

# 7. UX Requirements

## Platform
Static site untuk GitHub Pages.

## Layout
Mobile-first single page.

### Breakpoints
- Mobile: `360px–767px`
- Tablet: `768px–1023px`
- Desktop: `1024px+`

## Header
- Sticky optional
- Logo kiri
- CTA WhatsApp kanan
- Mobile menu jika nav panjang
- Header tetap ringan, tidak menutupi konten

## Hero
- Headline jelas di atas fold mobile
- CTA WhatsApp terlihat tanpa scroll panjang
- Secondary CTA scroll ke kategori
- Visual perangkat kantor mendukung, bukan mengganggu

## Category cards
- Grid 1 kolom mobile
- Grid 2 kolom tablet
- Grid 3 kolom desktop
- Icon line simple per kategori

## Contact
- WhatsApp button besar
- Nomor bisa diklik
- Suggested message bisa disalin
- Jangan pakai form jika tidak ada backend

## CTA behavior
Primary WhatsApp link buka tab / app:

```html
<a href="https://wa.me/[nomor-disamarkan]?text=Halo%20Raihan%20Com%2C%20saya%20ingin%20menanyakan%20alat%20bekas%20kantor.%20Barang%3A%20%5Bjenis%20barang%5D.%20Kondisi%3A%20%5Bkondisi%5D.%20Lokasi%3A%20%5Blokasi%5D.%20Saya%20bisa%20kirim%20foto." target="_blank" rel="noopener">
  Hubungi via WhatsApp
</a>
```

## Performance
- Static HTML/CSS/JS ringan
- Hindari dependency berat
- Optimalkan gambar
- Lazy load gambar non-hero
- Gunakan font system atau maksimal 1 webfont

## No fabricated content
Tidak boleh menampilkan:
- Statistik palsu
- Testimonial palsu
- Logo klien palsu
- Klaim “terpercaya sejak…” jika tahun tidak diketahui
- Alamat jika tidak tersedia
- Area layanan jika tidak tersedia
